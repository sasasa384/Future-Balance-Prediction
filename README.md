# 未来予測残高システム

## 概要
将来の収支・残高を予測できる、個人向け家計管理Webアプリです。
大学の授業(ソフトウェア開発演習)における、チーム開発プロジェクトの一環として制作しました。

ユーザーが日々の収入・支出を記録すると、将来の予測残高をカレンダー上で確認できる機能を中心に開発しています。

## 主な機能
- 収入・支出の記録
- 未来予測残高の算出・表示
- カレンダー形式での残高表示
- シフト・バイト先の管理
- 収支管理
- 初期設定

## 使用技術
- HTML
- CSS
- JavaScript

## 私の担当箇所
本プロジェクトでは、**カレンダー表示機能・予測残高計算**の設計・実装を担当しました。

- [Qiitaの参考記事](https://qiita.com/kandai/items/b1850750b883f83b9bee)を参考に、カレンダーの描画・月送り機能を実装
- 参考記事の基本機能に加えて、以下の独自機能を追加実装
  - 予測残高の計算(`getBalance()` 関数):基準日の残高を起点に、指定日までの収入を加算・支出を減算して算出
  - 日付タップ時の過去/未来判定:過去の日付は「確定残高」、未来の日付は「予測残高」として表示を切り替え
  - 今日の日付の強調表示
  - ドット表示機能:シフト担当メンバーが実装した `state.shifts` のデータ構造と連携し、シフト・収入・支出の有無をそれぞれ紫・緑・赤のドットでカレンダー上に表示
- テストはダミーデータを `state` に直接入力し、ブラウザ上で日付ごとの予測残高の変化・ドット表示の正確性を目視で確認

### 工夫した点・苦労した点
担当箇所自体は基本的な内容だったが、今日の日付の強調表示やドット表示など初めて実装する機能が多く、設計方針を固める段階で時間がかかった。

## 開発中に直面した課題と学び
- 基準日当日の収支が残高に反映されないバグが発生。`getBalance()` 内の比較演算子が `>` になっており、当日分が計算対象外だったことが原因。`>=` に修正して解決。
- 当日の残高が「予測残高」と誤表示されるバグが発生。日付比較を `Date` 型で行っていたことによる時刻のズレが原因。文字列比較に変更して解決。
- チームでコード統合した際、自分の実装したカレンダー機能が反映されておらず、予測残高が表示されない状態になっていたことがあった。統合担当メンバーとの連携不足が原因。
